# Claude Code 사용 팁

## 효율적인 프롬프트

- **구체적으로 작성**: "버그 고쳐줘"보다 `auth.ts:42`에서 토큰 만료 처리 버그처럼 파일/라인을 지정
- **컨텍스트 제공**: 이미 시도한 것, 제약사항을 먼저 알려주기
- **병렬 처리**: 독립적인 작업은 한 메시지에 묶어서 요청

## 핵심 단축 명령

| 명령 | 설명 |
|---|---|
| `!<command>` | 쉘 명령 직접 실행 (결과가 대화에 포함됨) |
| `/<skill>` | 스킬 호출 (예: `/commit`, `/simplify`) |
| `/help` | 도움말 |
| `/fast` | 빠른 출력 모드 토글 |

## 파일 & 메모리

- **CLAUDE.md**: 프로젝트 루트에 두면 자동 로딩 — 규칙/컨벤션 명시
- **메모리 시스템**: "기억해줘"라고 하면 영구 저장, 다음 세션에서도 유지
- **피드백 저장**: 교정해주면 같은 실수를 반복하지 않음

## 서브에이전트 활용

- **Explore**: 코드베이스 광범위 탐색 (여러 쿼리가 필요할 때)
- **Plan**: 복잡한 구현 전략 설계
- **general-purpose**: 멀티스텝 리서치
- 독립적인 작업은 **병렬로** 여러 에이전트 동시 실행

## 도구 선택

- 파일 검색은 **Glob**, 내용 검색은 **Grep** (bash의 `find`/`grep` 대신)
- 읽기는 **Read**, 편집은 **Edit** — 전용 도구가 리뷰에 유리

## 안전한 작업

> ⚠️ 파괴적 작업(`push --force`, `rm -rf`, `reset --hard`)은 먼저 확인

- `--no-verify`로 훅 건너뛰지 않기 — 근본 원인을 수정
- 커밋은 **명시적으로 요청할 때만**

## 플랜 모드

- 큰 변경 전 `EnterPlanMode`로 계획을 먼저 승인받기
- 중간에 접근이 바뀌면 플랜 업데이트

## 확장 통합

- **IDE 확장**: VS Code, JetBrains 지원
- **MCP 서버**: Chrome 브라우저 자동화, Google Workspace 등
- **스케줄링**: `/schedule`로 cron 작업 등록
- **훅(hooks)**: `settings.json`에 등록해 자동화 트리거

## 워크트리

- `isolation: "worktree"`로 격리된 브랜치에서 실험 — 원본에 영향 없음

## 팁 오브 팁

1. 에이전트에겐 **"왜"까지 설명** — 단순 명령보다 맥락이 결과 품질을 좌우
2. 응답 길게 원치 않으면 **"간결히"**, **"200자 이내"** 명시
3. 이슈/PR 참조는 `owner/repo#123` 형식으로 클릭 가능
